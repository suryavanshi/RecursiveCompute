Architecture comparison · public information through July 2026

Four answers to the inference memory wall.

RCIF is closest to Groq in its pursuit of deterministic token execution, closest to Etched in its rack-level decode focus, and closest to Cerebras in treating memory locality as architecture—not an afterthought. Its distinct bet is cheap LPDDR6 capacity plus virtualized, reusable KV.

The short version

Groq puts weights in distributed SRAM and schedules every cycle. Etched co-designs low-voltage math with a shared HBM/SRAM memory domain. Cerebras turns a wafer into one enormous SRAM-rich dataflow processor. RCIF keeps weights and active KV in lower-cost local DRAM, then makes prefix locality and KV paging first-class.

Economic thesis
Capacity per dollar

Avoid HBM/2.5D packaging in the baseline; buy enough compute to saturate sustained local DRAM.

Latency per token

Spend silicon on SRAM, deterministic datapaths, and direct links; distribute model storage over chips.

FLOPs per watt at rack scale

Lower math voltage and reduce the HBM-versus-SRAM compromise through cluster co-design.

Communication avoided

Pay the wafer-scale manufacturing and system cost to collapse a cluster’s fine-grained communication onto silicon.

Strategic positioning

RCIF is not a smaller copy of any one competitor.

It selects one lesson from each architecture, then changes the memory economics and the software-visible object being optimized.

RCIF

Optimize the token’s locality over the life of an agent session.

  • LPDDR6 capacity first
  • Virtual, tiered, shareable KV
  • Hardware decode graph
  • Prefix-aware routing

Groq

Optimize a compiled tensor stream so every cycle and packet is known.

  • SRAM primary storage
  • Single logical core
  • Static compute + network
  • Low batch latency

Etched

Optimize an entire inference rack’s power, math, and shared memory together.

  • Low-voltage math
  • HBM/SRAM hybrid
  • Scale-up memory pool
  • Prefill + decode

Cerebras

Optimize locality by replacing many packaged chips with one wafer-scale fabric.

  • 44 GB on-chip SRAM
  • 900K dataflow cores
  • On-wafer mesh
  • Training + inference

What the comparison means

RCIF’s opportunity is also its proof burden.

The architecture is credible only if cheap memory capacity survives real access efficiency, and if session-level KV reuse offsets the latency advantage of SRAM-heavy competitors.

Where RCIF can differentiate

Neither raw SRAM bandwidth nor peak compute is the product claim.

  • More local weight and KV capacity per package dollar
  • Prefix reuse as a routing and hardware-memory primitive
  • Warm/cold KV tiers for long-lived agent sessions
  • Compute sized from measured sustained bytes

Where RCIF trails today

The public competitors have silicon or systems; RCIF has a roadmap and an early executable slice.

  • No paged-attention or tensor datapath yet
  • No physical LPDDR6 controller/PHY proof
  • No measured token-level performance or power
  • No demonstrated multi-chip collective engine

The decisive experiment

Show that a repeated-prefix coding workload stays local and meets tail latency at lower system cost.

  • Measure sustained LPDDR efficiency, not raw pins
  • Report KV bytes and tier hits per output token
  • Cap collective share of TPOT
  • Compare p95/p99 TPOT at equal model, quality, and batch

Evidence boundary

Target architecture is not current RTL.

The repo currently proves descriptor flow and early KV/DMA semantics. The competitive thesis remains a hypothesis until the decode datapath and system model close together.

Implemented now

A bounded, deterministic control-and-memory prototype:

  • Command/completion queueing and backpressure
  • Scheduler opcodes, counters, and error handling
  • KV map/translate with a reusable TLB/MMU
  • DMA descriptor validation
  • Page-backed gather/scatter SRAM model with checksum completion
  • Python and Modal-backed Verilator regression paths
